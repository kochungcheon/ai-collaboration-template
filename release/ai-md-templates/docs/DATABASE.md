# 데이터베이스 스키마 템플릿 (Database Schema Template)

> 도메인 모델을 테이블/컬렉션으로 정의하는 문서입니다. 필요 없는 섹션은 제거하고, 실제 필드를 채워 넣으세요.

## 1. ERD 개요
- 핵심 엔티티 목록과 관계를 간단히 서술합니다 (예: Users ⟷ Orders ⟷ Payments).
- 이미지나 다이어그램 링크가 있다면 첨부하세요.

## 2. 테이블/컬렉션 정의

### `<TableName>`
| Column | Type | Constraints | Description |
| --- | --- | --- | --- |
| `id` | BIGINT | PK, Auto Inc | ... |
| `created_at` | DATETIME | NOT NULL | ... |
| ... | ... | ... | ... |

- 인덱스, 파티셔닝, 샤딩 전략을 필요 시 기록합니다.

### `<AnotherTable>`
- 동일한 형식으로 반복합니다.

## 3. Reference Data & Lookup Tables
- Enum/Lookup 테이블을 정의하고 코드/설명/정렬 순서를 기록합니다.

## 4. 마이그레이션 전략
- 도구(Flyway, Liquibase 등), 스크립트 위치, 버전 네이밍 규칙.
- 데이터 백필이나 마이그레이션 순서를 정리합니다.

## 5. 데이터 품질 & 거버넌스
- 멱등성 키, 중복 방지 규칙, 감사 로그 요구사항 등을 명시하세요.
- 백업/복구 전략 및 보존 기간을 적습니다.
